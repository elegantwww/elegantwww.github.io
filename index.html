总结

选择

  1. github pages + hexo
  2. 支持markdown是必须的：typecho也是可以的

github 步骤

  1. 配置SSH keys: Git shell,
    $ $ cd ~/.ssh
  2. 生成新的SSH Key: 后回车
    $ $ ssh-keygen -t rsa -C “邮件地址@youremail.com”
  3. 打开C:\Users\Zing4.ssh下的id_rsa.pub
  4. 在web上的用户setting下的ssh keys下add:
  5. 测试
    $ $ ssh -T git@github.com
  6. 设置用户信息
    $ $ git config –global user.name “elegantwww”
    git config –global user.email “elegantwww@gmail.com”

hexo 步骤

  1. 安装 node.js
  2. node-v4.0.0-x64.msi
  3. a dll required for this install to complete could not be run.【解决用v0.10.40/x64/替代】
  4. 配置pages: 仓库名必须为【your_user_name.github.io】
  5. 安装hexo:
    $ $ npm install hexo

调试

  1. 新建hexo文件夹 C:\Users\Zing4\Documents\GitHub\hexo
  2. 初始化
    $ $ cd hexo
    $ $ hexo init #初始化
    $ $ npm install
    $ $ hexo server #启动服务
  3. 部署
    $ $ hexo new “这里是博客名字”

  4. 改 _config.yml

  5. 【错误 type: github】
    $ $npm install hexo-deployer-git –save
    修改配置文件
    将_config.yml中
    deploy 的 type由github改为git

新建文章

  1. 新建文章
    $ $ hexo new “文章题目”
  2. 页面
    $ $ hexo new page “1” #面面链接名称
  3. 生成静态文件
    $ $ hexo generate # 简写：hexo g
  4. 服务器
    $ $ hexo server # 简写：hexo s
  5. 部署到git
    $ $ hexo deploy # 简写：hexo d
